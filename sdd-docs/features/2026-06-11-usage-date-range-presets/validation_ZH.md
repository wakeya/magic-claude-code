# 使用统计快捷日期范围 — 验证清单

**功能：** 使用统计快捷日期范围
**状态：** planned
**最后更新：** 2026-06-11

## 验收标准

1. **位置：** 快捷日期条显示在使用统计二级页签和筛选条件块之间。
2. **默认范围：** 首次进入使用统计页面时默认选中 `近7天`。
3. **近7天语义：** `近7天` 不包含今天，结束日期为昨天，开始日期为昨天往前 6 天。
4. **近30天语义：** `近30天` 不包含今天，结束日期为昨天，开始日期为昨天往前 29 天。
5. **今日语义：** `今日` 的开始日期和结束日期均为今天。
6. **同步刷新：** 点击任一快捷项会同步开始/结束日期，并刷新概览、请求日志、Provider、模型和 Usage 覆盖率数据。
7. **保留筛选：** 点击快捷项不会重置 Provider、模型、状态、Usage 来源、统计口径、来源入口和搜索词。
8. **分页行为：** 日期变化后请求日志回到第一页。
9. **手动匹配：** 手动日期正好匹配预设时，对应按钮高亮。
10. **手动自定义：** 手动日期不匹配任何预设时，三个按钮都不高亮。
11. **响应式：** 小屏幕下快捷日期条不发生文本溢出或控件重叠。

## 手动验证建议

### 场景 1：默认进入页面

1. 打开管理页面并进入“使用统计”。
2. 预期：快捷日期条显示在二级页签和筛选条件之间。
3. 预期：`近7天` 高亮。
4. 预期：开始日期和结束日期符合不含今天的 7 个完整自然日。

### 场景 2：切换快捷范围

1. 点击 `今日`。
2. 预期：开始日期和结束日期均为今天。
3. 点击 `近30天`。
4. 预期：结束日期为昨天，开始日期为昨天往前 29 天。
5. 预期：统计数据重新加载。

### 场景 3：手动日期

1. 手动输入一个不匹配任何预设的日期范围。
2. 预期：三个快捷按钮都不高亮。
3. 手动输入一个正好等于 `近7天` 的日期范围。
4. 预期：`近7天` 高亮。
