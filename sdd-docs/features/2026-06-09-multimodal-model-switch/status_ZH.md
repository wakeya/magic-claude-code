## 概要

已为 Provider 增加可选“多模态切换”配置。请求中包含图片、PDF、音频或视频等非文本内容时，若 Provider 开启该开关并配置了多模态模型 ID，代理会在转发前将 `model` 覆盖为该多模态模型。未开启或纯文本请求继续沿用现有模型映射。

## 已完成

1. Provider 配置新增 `multimodal_switch` 和 `multimodal_model`。
2. 代理请求转换阶段递归检测非文本内容。
3. SQLite 存储和旧库补列逻辑已实现。
4. Admin API 创建、查询、更新、复制均支持多模态配置。
5. Provider 配置弹窗新增 `多模态切换`、问号提示和 `多模态模型 ID`。
6. Provider 卡片显示多模态切换摘要。
7. 桌面端 Provider 弹窗加宽至约供应商页面内容区的四分之三，提升模型映射和多模态配置编辑体验。
8. 自动化测试和前端构建通过。
9. 用户已使用真实 Mimo 多模态模型 `mimo-v2.5` 验证通过。

## 重新评估触发条件

1. 需要按原始模型分别配置不同多模态模型。
2. 多模态模型仍因 Anthropic Beta Header 或其他协议字段报错。
3. 出现新的非文本内容格式，当前递归检测未识别。
4. 请求体接近 10MB 上限且递归扫描对延迟有可观影响。
