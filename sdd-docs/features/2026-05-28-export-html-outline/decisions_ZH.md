# Decisions

## D-001: 每条 user 消息作为大纲条目

**日期：** 2026-05-28
**状态：** accepted

**背景：** 用户要求参考现有会话记录页面的实现方案。现有 SessionOutline.vue 即按每条 user 消息生成一个条目。

**决策：** 大纲中每条 user 消息对应一个条目，条目内容为该消息的预览（前 50 字符，去除多余空白，超过截断加 `...`）和时间戳。

**影响：** 实现简单、确定性高，与现有页面行为一致。缺点是当消息极长时条目仍较长。

**重新评估条件：** 用户反馈条目内容粒度不够细或过于冗长。

## D-002: IntersectionObserver 实现滚动高亮

**日期：** 2026-05-28
**状态：** accepted

**背景：** 需要在大屏和小屏导出的 HTML 中实现滚动时自动高亮当前大纲条目。

**决策：** 使用原生 `IntersectionObserver` API 监听所有 `#msg-*` 元素，动态更新大纲 active 状态。

**影响：** 性能好（不依赖 scroll 事件）、无外部依赖。缺点是 IE 不支持，但导出的 HTML 面向现代浏览器。

**重新评估条件：** 目标浏览器需要支持 IE。
