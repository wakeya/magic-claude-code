# Requirements

## 1. Overview

为导出的 HTML 会话文件增加右侧大纲导航面板和回到顶部按钮，提供与在线会话记录页面一致的浏览体验。

### Goals

- 导出的 HTML 文件是完全自包含的静态文件（无外部依赖，可离线打开）
- 大纲展示每条 user 消息的摘要，点击可跳转，滚动时自动高亮当前条目
- 响应式设计：大屏固定右侧面板，小屏弹窗覆盖

### Non-Goals

- 不改动现有的会话记录页面（SessionBrowser.vue）
- 不引入任何外部依赖（CDN、框架等）

---

## 2. Scope

### In Scope

- 修改 `internal/session/export.go`，在导出模板中增加大纲面板 HTML/CSS/JS
- 大纲数据在 Go 端构建，注入模板
- 每条消息 `<section>` 添加 `id="msg-{index}"` 用于锚点跳转
- 响应式：大屏（≥1024px）右侧固定面板，小屏（<1024px）弹窗
- 主题跟随导出时传入的 `theme` 参数

### Out of Scope

- 会话记录页面的改动
- 导出 PDF 或其他格式
- 服务端实时预览功能

---

## 3. Acceptance Criteria

1. 导出的 HTML 文件无外部依赖，可直接双击在浏览器打开
2. 大纲面板显示所有 user 消息的预览（最多 50 字符 + `...`）
3. 点击大纲条目能平滑滚动到对应消息
4. 滚动页面时，大纲自动高亮当前可见的消息条目
5. 回到顶部按钮始终可用
6. 在 1024px 及以上宽度时显示右侧面板，以下宽度时显示弹窗
7. 主题（亮/暗）跟随导出时传入的 `theme` 参数
8. 文件大小增长控制在合理范围内（预计增加 <20KB）

---

## 4. Constraints

- 必须复用现有的 CSS 变量体系（`--surface`, `--text`, `--accent`, `--border` 等）
- 必须复用现有的 `data-theme` 属性机制
- Go 模板中嵌入的 JS 必须是原生 JS，无框架依赖
- 不使用 Tailwind，必须手写内联 CSS
