# Provider 用量查询弹窗设计

## 目标

将 Provider 用量配置从独立页面改为与现有 Provider 编辑体验一致的弹窗。供应商列表保留在半透明遮罩后方，弹窗内继续并排展示用量配置和最新查询结果。

## 范围

本次仅调整前端展示和交互编排。现有用量 API、后台调度、凭据语义、查询适配器及 snapshot 存储均保持不变。

## 组件架构

### `ProviderUsageModal.vue`

新增一个不依赖路由的组件，负责用量表单和查询状态。它接收：

- `providerId`：正在编辑用量配置的 Provider ID。
- `providerName`：弹窗标题中显示的 Provider 名称。

它发出：

- `close`：用户通过取消、遮罩、关闭按钮或 Escape 请求关闭。
- `saved`：配置已保存；启用用量查询时传出最新 snapshot，禁用时传出 `null`。

组件复用 `QuotaResultDisplay`、`quotaForm` 工具、`useApi` 和现有 i18n 文案，不再使用 `useRoute` 或 `useRouter`。

### `DashboardView.vue`

Dashboard 持有 `usageProviderId`。供应商卡片触发“用量”事件时，只设置该 ID 并异步加载 `ProviderUsageModal`，不执行导航，也不改变 URL。

收到 `saved` 后，Dashboard 更新 Provider snapshot；载荷为 `null` 时则移除该 snapshot。随后清空 `usageProviderId`，并把焦点还给打开弹窗的“用量”按钮。普通关闭时同样清空 ID 并重新加载 snapshot，保证弹窗内执行过的手动刷新能反映到供应商卡片。

### 旧路由兼容

继续接受 `/providers/:providerId/usage` 作为兼容入口。路由将其重定向为 `/?tab=providers&usage_provider=<id>`。Dashboard 读取该一次性参数并打开弹窗，随后立即通过 `router.replace` 移除 `usage_provider`，最终保留稳定的供应商列表 URL。

迁移完逻辑后删除不再使用的独立页面 `ProviderUsageView.vue`。

## 布局与视觉设计

弹窗以 `ProviderModal` 作为视觉规范来源，不引入新的配色或字体：

- 半透明黑色遮罩和居中的 `app-panel` 面板。
- 宽度约 `90vw`，最大宽度 `1180px`，最大高度 `90vh`。
- 复用现有边框、圆角、间距、控件、字体、主题及按钮样式。
- 标题为“用量查询 · <Provider 名称>”，右上角提供关闭按钮。
- 桌面端（`lg` 及以上）采用双栏：左侧配置，右侧最新结果和“立即刷新”。
- 小屏幕改为纵向堆叠，配置在前、结果在后，不产生横向滚动。
- 底部操作为“取消、测试查询、保存”。

不新增设计系统颜色或字体，继续以应用现有明暗主题为准。

## 交互模型

- 打开弹窗后显示加载状态，配置和 snapshot 就绪后再展示内容。
- 取消、点击遮罩、关闭按钮或 Escape 直接关闭，不提示未保存内容，与 `ProviderModal` 一致。
- “测试查询”使用当前未保存表单执行临时查询，显示临时结果并保持弹窗打开。
- “立即刷新”执行正式查询并重新加载已保存 snapshot，保持弹窗打开。
- 每个进行中的操作只禁用对应按钮，防止重复提交。

### 保存、查询与关闭顺序

“保存”根据保存后的启用状态执行不同流程。

启用用量查询时：

1. 构建并提交用量配置。
2. 保存成功后清空已提交的敏感输入，并重置清除凭据标志。
3. 立即使用已保存配置执行一次正式用量查询。
4. 只有查询响应和规范化结果都报告成功，才视为查询成功。
5. 重新读取 Provider 用量响应，取得已持久化的 snapshot。
6. 通过 `saved` 传出 snapshot，更新供应商卡片，并在与 `ProviderModal` 一致的短暂成功延迟（约 800ms）后关闭弹窗。

如果保存配置失败，弹窗保持打开并显示保存错误。如果配置保存成功但正式查询失败，已保存的配置继续保留，弹窗保持打开，并明确提示“配置已保存，但查询失败”。这样可以避免关闭后供应商卡片没有当前结果。

禁用用量查询时，保存后跳过正式查询，通过 `saved` 传出 `null`，移除该 Provider 的 snapshot 展示，并在相同成功延迟后关闭。禁用配置不得被误报为查询失败。

## 错误与空状态

- Provider ID 无效时显示本地化的“供应商不存在”状态，并提供关闭操作。
- 初始加载失败必须显示，不再静默忽略。
- 保存、测试和刷新失败分别显示在对应操作或结果区域。
- 没有 snapshot 时显示现有“从未查询”状态。
- 保存失败时不得触发正式查询。
- 保存后的正式查询失败时不得发出 `saved`，也不得自动关闭。

## 可访问性

- 面板使用 `role="dialog"`、`aria-modal="true"` 和关联标题的 `aria-labelledby`。
- 打开后焦点进入弹窗，关闭后返回原“用量”按钮。
- 没有破坏性确认操作时，Escape 可关闭弹窗。
- 所有字段保留可见标签和焦点样式。
- 操作按钮使用语义化元素，并在请求期间暴露禁用状态。
- 弹窗打开期间禁止背景页面滚动。

## 测试

定向前端测试覆盖：

- 点击 Provider“用量”按钮打开弹窗且 URL 不变。
- 弹窗收到正确的 Provider ID 和名称。
- 弹窗具备共享遮罩/面板结构、桌面双栏、移动端堆叠及 dialog 属性。
- 保存时先更新配置，再执行正式查询。
- 保存和查询均成功时传出最新 snapshot，并在成功延迟后关闭。
- 保存禁用配置时跳过查询、移除卡片 snapshot，并在成功延迟后关闭。
- 保存失败和保存后查询失败时，弹窗保持打开并显示不同错误。
- “测试查询”和“立即刷新”不会关闭弹窗。
- 普通关闭会重新加载 Dashboard snapshot 并恢复焦点。
- 旧路由能打开弹窗并移除一次性参数。

验证命令包括完整前端测试、前端生产构建和 Go 测试套件。本次预期不修改后端。
