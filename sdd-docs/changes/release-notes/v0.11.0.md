## v0.11.0 更新内容

### Security
- **请求参数日志脱敏（CWE-532）**：上游返回 ≥400 时，`[Proxy] Error` 日志的请求摘要改为类型安全白名单——只保留 `model`、`stream`、数值生成参数（`max_tokens`/`max_output_tokens`/`temperature`/`top_p`/`top_k`）与集合计数（`messages`/`tools`/`input`）。`system` 系统提示词、`metadata`、凭据、未知扩展与消息正文不再写入进程日志

### Fixed
- **SSE-labeled HTTP 错误不再静默丢失**：上游以 `Content-Type: text/event-stream` 返回 4xx/5xx 时，原先被 SSE 心跳路径抢先，错误不记日志。现改为先判状态码再判 SSE（`StatusCode < 400 && isSSEStream`），此类错误正确落到 `[Proxy] Error` 日志

### Changed
- **结构化协议诊断**：错误日志的请求摘要升级为结构化协议摘要——`stream` 区分缺失/false/true；`messages` 细化到角色（user/assistant）计数与 content block 类型计数；`tools` 记录名称摘要与 schema 关键字（`$schema`/`additionalProperties`/`format` 等）出现次数。仅保留定位上游协议兼容问题所需的结构信息，不含 prompt、正文或凭据

### Docs
- SSE-labeled HTTP 错误处理 feature spec（中英双语）：`sdd-docs/features/2026-06-30-non-2xx-sse-error-handling/`
- 智谱 Web 工具兼容性恢复 spec（中英双语）：`sdd-docs/features/2026-07-01-zhipu-web-tools-compat/`
- 安全审查记录（中英双语，确认 CWE-532 闭合、无遗留缺陷）

### Known Issues
- 智谱 GLM 上游对 Claude Code 动态加载 `WebFetch`/`WebSearch` 后的请求可能返回 `1210` 参数错误（与 `stream` 字段缺失触发的 fallback 路径相关）。本版提供结构化诊断能力便于定位，recovery 实现留待后续版本

## v0.11.0 Release Notes

### Security
- **Request-parameter log redaction (CWE-532)**: When the upstream returns ≥400, the request summary in the `[Proxy] Error` log switches to a type-safe allowlist — keeping only `model`, `stream`, numeric generation params (`max_tokens`/`max_output_tokens`/`temperature`/`top_p`/`top_k`), and collection counts (`messages`/`tools`/`input`). `system` prompts, `metadata`, credentials, unknown extensions, and message bodies are no longer written to process logs

### Fixed
- **SSE-labeled HTTP errors no longer silently lost**: When the upstream returns 4xx/5xx with `Content-Type: text/event-stream`, the error previously got intercepted by the SSE-heartbeat path and wasn't logged. Now the status code is checked before SSE (`StatusCode < 400 && isSSEStream`), and such errors correctly land in the `[Proxy] Error` log

### Changed
- **Structured protocol diagnostics**: The request summary in error logs is upgraded to a structured protocol summary — `stream` distinguishes missing/false/true; `messages` is refined to role (user/assistant) counts and content-block-type counts; `tools` records name digests and schema-keyword (`$schema`/`additionalProperties`/`format`, etc.) occurrence counts. Only the structural info needed to diagnose upstream protocol-compat issues is retained — no prompts, bodies, or credentials

### Docs
- SSE-labeled HTTP error-handling feature spec (bilingual): `sdd-docs/features/2026-06-30-non-2xx-sse-error-handling/`
- Zhipu web-tool compatibility recovery spec (bilingual): `sdd-docs/features/2026-07-01-zhipu-web-tools-compat/`
- Security review record (bilingual, confirming CWE-532 closure and no residual defects)

### Known Issues
- The Zhipu GLM upstream may return a `1210` parameter error for requests after Claude Code dynamically loads `WebFetch`/`WebSearch` (related to the fallback path triggered by the missing `stream` field). This version provides structured diagnostics for locating it; recovery is deferred to a later version
