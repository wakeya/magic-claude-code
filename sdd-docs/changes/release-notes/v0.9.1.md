## v0.9.1 更新内容

### Added
- **供应商导入导出**：供应商管理页新增 JSON 导入导出，支持多主机间批量迁移供应商配置。每张卡片左上角新增复选框，工具栏新增全选/导出/导入按钮
  - **导出**：选中供应商 → 下载 `providers-export-YYYYMMDD-HHMMSS.json`（含真实 API Token，下载前弹窗确认安全风险）
  - **导入**：选择 JSON 文件 → 客户端预览（新增/冲突计数）→ 选择冲突策略（跳过/覆盖/复制，默认跳过）→ 确认导入 → 刷新列表
  - **冲突策略**：`skip`（保留现有）、`overwrite`（替换同 ID 字段，保留 created_at）、`duplicate`（冲突项生成新 ID 追加，非冲突项保留原 ID）
  - **后端 API**：`POST /api/providers/export` 按 ID 导出含真实 token 的 JSON；`POST /api/providers/import` 单次 Load→Save 周期应用，version 校验，无效供应商跳过并计入 errors，文件内 ID 去重
  - **安全**：导出/导入接口均位于 authMiddlewareFunc 之后；显式拒绝非 POST 方法（405）；前端校验 res.ok 并 try/catch 显示错误
  - **全选控件**：标题左侧三态复选框（全选/部分选中 indeterminate/未选）

### Fixed
- 导入版本校验：前端解析时校验 `version === 1`，缺失或非 1 在预览前拒绝，不绕过后端版本拒绝逻辑
- 导出失败提示：新增 `providers.export_failed` 文案，不再误用导入格式错误文案
- duplicate 策略语义：修正为仅冲突项生成新 ID，非冲突项保留原 ID 正常导入（跨机器同步不丢失原 provider ID）

### Docs
- 新增供应商导入导出功能 spec（中英双语，5/5 任务已完成并验证）：`sdd-docs/features/2026-06-23-provider-import-export/`

## v0.9.1 Release Notes

### Added
- **Provider import/export**: The provider-management page adds JSON import/export for batch-migrating provider configs across hosts. Each card gains a top-left checkbox; the toolbar adds select-all/export/import buttons
  - **Export**: Select providers → download `providers-export-YYYYMMDD-HHMMSS.json` (contains real API tokens; a confirmation dialog warns of the security risk before download)
  - **Import**: Choose a JSON file → client-side preview (new/conflict counts) → choose a conflict strategy (skip/overwrite/duplicate, default skip) → confirm import → refresh the list
  - **Conflict strategies**: `skip` (keep existing), `overwrite` (replace same-ID fields, preserve created_at), `duplicate` (generate new IDs for conflicts and append; non-conflicts keep original IDs)
  - **Backend API**: `POST /api/providers/export` exports JSON with real tokens by ID; `POST /api/providers/import` applies changes in a single Load→Save cycle, with version validation, invalid-provider skip into errors, and in-file ID dedup
  - **Security**: Both export/import sit behind authMiddlewareFunc; non-POST methods are explicitly rejected (405); the frontend checks res.ok and try/catch displays errors
  - **Select-all control**: A tri-state checkbox (all/some-indeterminate/none) at the title's left

### Fixed
- Import version validation: The frontend validates `version === 1` during parse; missing or non-1 is rejected before preview, not bypassing the backend's version rejection
- Export-failure copy: Added `providers.export_failed` copy instead of misusing the import format-error copy
- duplicate-strategy semantics: Corrected to generate new IDs only for conflicts; non-conflicts keep their original IDs (cross-machine sync doesn't lose original provider IDs)

### Docs
- New provider-import-export feature spec (bilingual, 5/5 tasks done and verified): `sdd-docs/features/2026-06-23-provider-import-export/`
