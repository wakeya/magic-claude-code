## v0.8.1 更新内容

### Fixed
- **Windows 引导乱码修复**：透明模式引导阶段 `certutil -addstore`、`setx` 等子进程在中文 Windows 上按活动代码页（GBK/CP936）输出本地化文本，被 exec 捕获后直接拼进错误信息会产生乱码（例如 CA 信任安装权限不足时的中文提示）。新增 `decodeCmdOutput`：在 Windows 上把 GBK 解码为 UTF-8；已是合法 UTF-8 的输出不误转；解码失败安全回退到原始字节，绝不丢失信息。非 Windows 平台行为不变。

## v0.8.1 Release Notes

### Fixed
- **Windows bootstrap garbled-output fix**: During transparent-mode bootstrap, child processes like `certutil -addstore` and `setx` emit localized text in the active code page (GBK/CP936) on Chinese Windows; capturing their output via exec and concatenating it directly into error messages produced mojibake (e.g. the Chinese prompt when CA-trust install lacks permission). Added `decodeCmdOutput`: on Windows it decodes GBK to UTF-8; output that is already valid UTF-8 is not mis-converted; on decode failure it safely falls back to the raw bytes, never losing information. Non-Windows behavior is unchanged.
