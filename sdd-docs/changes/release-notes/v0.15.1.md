## v0.15.1 Release Notes

### Security
- **Upgraded echarts 6.0.0 → 6.1.0 to fix moderate XSS** (GHSA-fgmj-fm8m-jvvx): a cross-site scripting vulnerability in admin-panel chart rendering, affecting the production frontend.
- **Upgraded vite 8.0.8 → 8.1.4 to fix high vulnerabilities** (GHSA-v6wh-96g9-6wx3 + GHSA-fx2h-pf6j-xcff): only affects the Windows dev-server, not the production build artifacts.
- **npm audit**: 2 vulnerabilities → 0.

### Changed
- **release.sh supports backfilling historical versions via RELEASE_REF**: New `RELEASE_REF` env var — defaults to `main` (unchanged behavior); when set to `RELEASE_REF=<tag>`, checks out that tag to build a genuine historical-version binary. The original script forced `checkout main`, so backfilling an old version would build on the latest code while injecting the old version number, producing incorrect artifacts.

### Docs
- npm audit fix feature spec and review notes (bilingual): `sdd-docs/features/2026-07-10-npm-audit-fix/`
