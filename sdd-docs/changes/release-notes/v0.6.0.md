## v0.6.0 更新内容

### Changed
- **发布方式迁移至 Release API**：Gitee/GitCode 二进制不再存入 git 仓库，通过 Release API 上传为附件，与 GitHub 保持一致
- **自动更新器改用 Release 下载 URL**：Gitee/GitCode 从 raw 文件 URL 切换到 Release download URL，免认证匿名下载
- **移除 AssetHeaders**：Release 下载无需认证头，简化 Updater 的 headers 传递逻辑
- **重写 `scripts/release.sh`**：新增 Gitee/GitCode Release 附件上传（multipart POST / OBS 两步式），GitLab Release 附 GitHub 下载链接
- **移除 `dist/release/` 目录**：构建改用临时目录，不再在 git 中保留二进制或 `.gitkeep`
- 移除已废弃的 CI 配置（`.gitlab-ci.yml`、`.workflow`、`.gitcode/workflows`）

### Fixed
- GitCode 下载 URL 使用 `gitcode.com` 域名而非 `api.gitcode.com`（后者返回 404）

## v0.6.0 Release Notes

### Changed
- **Release distribution migrated to the Release API**: Gitee/GitCode binaries are no longer committed to git; they are uploaded as Release attachments, aligning with GitHub
- **Auto-updater switched to Release download URLs**: Gitee/GitCode moved from raw-file URLs to Release download URLs, enabling anonymous unauthenticated downloads
- **AssetHeaders removed**: Release downloads need no auth headers, simplifying the updater's header-passing logic
- **`scripts/release.sh` rewritten**: Adds Gitee/GitCode Release attachment upload (multipart POST / OBS two-step); GitLab Release attaches GitHub download links
- **`dist/release/` directory removed**: Builds use a temp directory; no binaries or `.gitkeep` are kept in git
- Removed deprecated CI configs (`.gitlab-ci.yml`, `.workflow`, `.gitcode/workflows`)

### Fixed
- GitCode download URL uses the `gitcode.com` domain instead of `api.gitcode.com` (the latter returned 404)
