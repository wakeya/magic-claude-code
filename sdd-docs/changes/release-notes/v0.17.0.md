## v0.17.0 更新内容

### Added
- **代理日志显示暴露模型 Label 替代内部 em- ID**：通过 `/model` 切换到暴露模型后，请求/响应日志的 `model=` 字段此前记录的是 `ExposedModel.ID`（随机 `em-<hex>`，设计上无语义），现改为显示用户配置的 `ExposedModel.Label`，保留 `-> BackendModel` 映射结构。`config.ModelRoute` 新增 `ExposedLabel` 字段，`ResolveRoute` 命中暴露模型时填充；代理日志层提取纯函数 `formatModelLog` 做展示替换，仅影响日志可读性，不改变路由、请求体、failover 或 usage 记录。Label 与 BackendModel 同名时折叠为单 token。

### Fixed
- **修复 admin provider 测试在沙箱环境的环境敏感失败**：`TestSuccessfulProviderTestClearsCredentialFailure` 此前用 `example.com` 作为测试 APIURL，在沙箱 DNS 把 `example.com` 解析到 ULA 私有 IPv6（`fd00::/8`）时，`isInternalIP` 的 SSRF 检查误拦，导致 `handleTestProviderByID` 提前返回、到不了凭据恢复钩子。改用 RFC 5737 TEST-NET-3 文档保留 IP（`203.0.113.1`）作为测试 APIURL，`isReservedIP` 永不误判且无需 DNS 解析。

### Docs

## v0.17.0 Release Notes

### Added
- **Proxy logs show ExposedModel.Label instead of the internal `em-` ID**: After switching to an exposed model via `/model`, the request/response log's `model=` field previously recorded the `ExposedModel.ID` (random `em-<hex>`, opaque by design); it now shows the user-configured `ExposedModel.Label`, keeping the `-> BackendModel` mapping. `config.ModelRoute` gains an `ExposedLabel` field populated by `ResolveRoute` on exposed-model hits; the proxy log layer extracts a pure `formatModelLog` helper for the display substitution, affecting only log readability — routing, request body, failover, and usage recording are unchanged. Collapses to a single token when Label equals BackendModel.

### Fixed
- **Fix environment-sensitive admin provider test failure**: `TestSuccessfulProviderTestClearsCredentialFailure` used `example.com` as the test APIURL; in sandboxes where DNS resolves `example.com` to a ULA private IPv6 (`fd00::/8`), `isInternalIP`'s SSRF check falsely blocked it, making `handleTestProviderByID` return early before reaching the credential-recovery hook. Switched to the RFC 5737 TEST-NET-3 documentation IP (`203.0.113.1`), which `isReservedIP` never flags and requires no DNS lookup.
