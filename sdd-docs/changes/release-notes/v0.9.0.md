## v0.9.0 更新内容

### Added
- **监听地址可配置**：代理服务（proxy）和配置服务（admin）的监听地址和端口现在支持通过 CLI flag（`-proxy-listen`/`-proxy-port`/`-admin-listen`/`-admin-port`）、环境变量（`MCC_PROXY_LISTEN_ADDR`/`MCC_PROXY_PORT`/`MCC_ADMIN_LISTEN_ADDR`/`MCC_ADMIN_PORT`）和配置文件三层覆盖，优先级 flag > env > config，默认行为与升级前完全一致（`0.0.0.0:443` + `0.0.0.0:8442`）；前端只读展示实际监听地址，无编辑入口
- **CLI 本地化帮助与版本号**：`mcc -h` / `--help` 按系统语言显示中文/英文 flag 说明，`mcc -v` / `--version` 打印版本并退出（不启动服务）
- **`/api/status` 暴露监听字段**：新增 `proxy_listen_addr`、`proxy_port`、`admin_listen_addr`、`admin_port`、`gateway_listen_addr`、`gateway_listen_port` 六个字段，反映 CLI/env/config 解析后的实际生效值
- **前端监听状态只读区块**：状态概览区新增"监听状态"区块，展示三个服务的地址:端口，附操作风险提示（非 443 需端口转发、127.0.0.1 仅本机可达）
- **IPv6 地址归一化**：`normalizeListenAddr` 统一剥离 RFC 2732 方括号（`[::1]` → `::1`），适用于 proxy/admin/gateway 三个监听地址字段

### Fixed
- **启动失败立即退出**：proxy/admin/gateway 服务监听失败（端口占用、权限不足）现在通过 `startupErr` 通道触发 `log.Fatalf` 退出进程，不再仅记日志后以"部分服务可用"状态继续运行
- **Gateway 热重启不误杀进程**：gateway goroutine 过滤 `http.ErrServerClosed`（`Shutdown` 的正常返回），避免 `/api/config` 触发热重启后主进程误退出
- **IPv6 地址拼接全面修复**：`fmt.Sprintf("%s:%d", ...)` 全部替换为 `net.JoinHostPort`（启动、handler restart、bootstrap 指令、前端展示），消除 `::1:443` 和 `http://::1:17487` 等非法格式
- **前端 IPv6 防御性格式化**：`formatListenAddress` 先剥离已有括号再按需添加，防止后端返回 `[::1]` 时重复显示 `[[::1]]:port`

### Changed
- **推荐 hook 改用原生 `rtk hook claude`**：前端"推荐配置"的 hooks 命令从 `$HOME/.claude/hooks/rtk-rewrite.sh`（依赖 bash + jq）改为 `rtk hook claude`（原生二进制子命令），Linux/macOS/Windows 通用；config_path_note 中英文文案同步更新，移除"rtk 在 Windows 上 hook 可能不生效"的过时结论
