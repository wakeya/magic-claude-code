## v0.9.0 更新内容

### Added
- **监听地址可配置**：代理服务（proxy）和配置服务（admin）的监听地址和端口现在支持通过 CLI flag（`-proxy-listen`/`-proxy-port`/`-admin-listen`/`-admin-port`）、环境变量（`MCC_PROXY_LISTEN_ADDR`/`MCC_PROXY_PORT`/`MCC_ADMIN_LISTEN_ADDR`/`MCC_ADMIN_PORT`）和配置文件三层覆盖，优先级 flag > env > config，默认行为与升级前完全一致（`0.0.0.0:443` + `0.0.0.0:8442`）；前端只读展示实际监听地址，无编辑入口
- **CLI 本地化帮助与版本号**：`mcc -h` / `--help` 按系统语言显示中文/英文 flag 说明，`mcc -v` / `--version` 打印版本并退出（不启动服务）
- **`/api/status` 暴露监听字段**：新增 `proxy_listen_addr`、`proxy_port`、`admin_listen_addr`、`admin_port`、`gateway_listen_addr`、`gateway_listen_port` 六个字段，反映 CLI/env/config 解析后的实际生效值
- **前端监听状态只读区块**：状态概览区新增"监听状态"区块，展示三个服务的地址:端口，附操作风险提示（非 443 需端口转发、127.0.0.1 仅本机可达）
- **IPv6 地址归一化**：`normalizeListenAddr` 统一剥离 RFC 2732 方括号（`[::1]` → `::1`），适用于 proxy/admin/gateway 三个监听地址字段

### Fixed
- **启动失败立即退出**：proxy/admin/gateway 服务监听失败（端口占用、权限不足）现在通过 `startupErr` 通道触发 `log.Fatalf` 退出进程，不再仅记日志后以"部分服务可用"状态继续运行
- **Gateway 热重启不误杀进程**：gateway goroutine 过滤 `http.ErrServerClosed`（`Shutdown` 的正常返回），避免 `/api/config` 触发热重启后主进程误退出
- **IPv6 地址拼接全面修复**：`fmt.Sprintf("%s:%d", ...)` 全部替换为 `net.JoinHostPort`（启动、handler restart、bootstrap 指令、前端展示），消除 `::1:443` 和 `http://::1:17487` 等非法格式
- **前端 IPv6 防御性格式化**：`formatListenAddress` 先剥离已有括号再按需添加，防止后端返回 `[::1]` 时重复显示 `[[::1]]:port`

### Changed
- **推荐 hook 改用原生 `rtk hook claude`**：前端"推荐配置"的 hooks 命令从 `$HOME/.claude/hooks/rtk-rewrite.sh`（依赖 bash + jq）改为 `rtk hook claude`（原生二进制子命令），Linux/macOS/Windows 通用；config_path_note 中英文文案同步更新，移除"rtk 在 Windows 上 hook 可能不生效"的过时结论

### Docs
- 新增监听地址配置功能 spec（中英双语）：`sdd-docs/features/2026-06-23-listen-address-config/`
- 新增 bootstrap IPv6 URL 回归测试（zh/en 双语断言 `http://[::1]:17487`）

## v0.9.0 Release Notes

### Added
- **Configurable listen addresses**: The proxy and admin services' listen addresses and ports now support three-layer override via CLI flags (`-proxy-listen`/`-proxy-port`/`-admin-listen`/`-admin-port`), env vars (`MCC_PROXY_LISTEN_ADDR`/`MCC_PROXY_PORT`/`MCC_ADMIN_LISTEN_ADDR`/`MCC_ADMIN_PORT`), and config file. Priority is flag > env > config; defaults match pre-upgrade behavior exactly (`0.0.0.0:443` + `0.0.0.0:8442`). The frontend shows the effective listen addresses read-only, with no editor.
- **CLI localized help and version**: `mcc -h` / `--help` shows Chinese/English flag descriptions by system language; `mcc -v` / `--version` prints the version and exits without starting the service
- **`/api/status` exposes listen fields**: Six new fields — `proxy_listen_addr`, `proxy_port`, `admin_listen_addr`, `admin_port`, `gateway_listen_addr`, `gateway_listen_port` — reflecting the effective values after CLI/env/config resolution
- **Frontend read-only listen-status block**: The status overview adds a "Listen Status" block showing the address:port of all three services, with operational-risk notes (non-443 needs port forwarding; 127.0.0.1 is localhost-only)
- **IPv6 address normalization**: `normalizeListenAddr` strips RFC 2732 brackets uniformly (`[::1]` → `::1`), applying to all three listen-address fields

### Fixed
- **Startup failure exits immediately**: proxy/admin/gateway listen failures (port in use, insufficient permission) now trigger `log.Fatalf` via the `startupErr` channel, instead of continuing with partial services after only logging
- **Gateway hot-restart no longer kills the process**: The gateway goroutine filters `http.ErrServerClosed` (the normal return of `Shutdown`), preventing the main process from exiting after `/api/config` triggers a hot restart
- **IPv6 address concatenation fixed broadly**: `fmt.Sprintf("%s:%d", ...)` is replaced everywhere with `net.JoinHostPort` (startup, handler restart, bootstrap directives, frontend display), eliminating invalid formats like `::1:443` and `http://::1:17487`
- **Frontend defensive IPv6 formatting**: `formatListenAddress` strips existing brackets before re-adding as needed, preventing duplicate `[[::1]]:port` display when the backend returns `[::1]`

### Changed
- **Recommended hook now uses native `rtk hook claude`**: The frontend's "recommended config" hooks command changes from `$HOME/.claude/hooks/rtk-rewrite.sh` (bash + jq) to `rtk hook claude` (native binary subcommand), universal across Linux/macOS/Windows; config_path_note copy updated in both languages, removing the outdated "rtk hooks may not work on Windows" note

### Docs
- New listen-address-config feature spec (bilingual): `sdd-docs/features/2026-06-23-listen-address-config/`
- New bootstrap IPv6 URL regression test (zh/en bilingual assertions for `http://[::1]:17487`)
