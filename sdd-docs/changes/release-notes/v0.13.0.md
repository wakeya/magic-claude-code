## v0.13.0 更新内容

### Added
- **Node.js 客户端 CA 信任自动配置**：Node.js 忽略操作系统 CA 信任库，系统级 CA 安装无法让 Claude Code（Node 客户端）信任 mcc 的 MITM 证书。新增 bootstrap 步骤将 `NODE_EXTRA_CA_CERTS` 持久化到用户环境——Windows 走 `setx`（用户级注册表）+ pwsh `$PROFILE` 兜底，macOS 走 `launchctl setenv` + shell profile，Linux 走 shell profile。通过 CA 指纹 marker 实现幂等；检测到用户自定义值时保留不覆盖；部分失败（setx/launchctl 失败但 profile 已写入）不标记、下次启动重试；Docker 容器内跳过
- **mcc.exe 应用图标**：Windows 二进制内嵌应用图标，任务栏 / 开始菜单显示品牌图标。新增 `make icon` 目标从 logo 自动重新生成（裁剪透明边距 + 230px 适配 256 画布，预留 5% 安全边距避免边缘裁切）

### Security
- **PowerShell 注入防护（P2-1）**：CA 路径渲染为 PowerShell 单引号字面量（home 相对路径走 `Join-Path`，绝对路径单引号包裹），拒绝 CR/LF 以阻断 `$()` / 反引号 / 换行命令注入
- **symlink 与特权运行防护（CWE-59 / P2-2）**：拒绝符号链接 profile 与 marker，防止跨路径篡改；检测到特权运行（管理员 / root）时拒绝 profile 变更，避免写入错误的用户配置
- **marker 身份绑定（F-4）**：NodeCA marker 绑定证书路径与用户身份，写入 / 读取均校验身份一致性，防止跨用户 marker 误用
- **fail-closed 校验链（F-1 / F-2）**：环境变更前先检查 pwsh / POSIX profile 的用户自定义值，有则跳过 `setx` / `launchctl` 不覆盖；profile 父链与文件系统根校验失败、profile 读取异常时 fail-closed；pwsh profile 部分失败返回 `ErrPartialSuccess`，不再因任一成功而掩盖整体失败

### Changed
- **Windows 环境变更广播**：`setx` 写入后广播环境变量变更，确保已运行进程能感知新的 `NODE_EXTRA_CA_CERTS`；Windows 客户端重启序列说明同步更新
- **旧版 pwsh profile 自动迁移**：升级场景下检测并清理旧版 `$mccCa` 残留行，一次性迁移到 mcc marker 块，恢复 marker + `setx` 持久化路径执行

### Docs
- Node.js 客户端 CA 信任自动配置 feature spec 与评审记录（中英双语）：`sdd-docs/features/2026-07-04-node-extra-ca-certs-auto-setup/`

## v0.13.0 Release Notes

### Added
- **Automatic CA trust setup for Node.js clients**: Node.js ignores the OS CA trust store, so a system-level CA install cannot make Claude Code (a Node client) trust mcc's MITM certificate. A new bootstrap step persists `NODE_EXTRA_CA_CERTS` to the user environment — Windows via `setx` (per-user registry) + pwsh `$PROFILE` fallback, macOS via `launchctl setenv` + shell profile, Linux via shell profile. Idempotent via a CA-fingerprint marker; user-custom values are detected and preserved (never overwritten); partial failures (setx/launchctl failed but profile written) are not marked and retry on next launch; skipped inside Docker containers.
- **mcc.exe application icon**: The Windows binary now embeds an application icon, showing branded artwork in the taskbar / start menu. A new `make icon` target regenerates it from the logo (trims transparent margins + fits content to 230px on a 256 canvas, leaving a 5% safety margin against edge clipping).

### Security
- **PowerShell injection hardening (P2-1)**: The CA path is rendered as a PowerShell single-quoted literal (`Join-Path` for home-relative paths, single-quoted absolute paths); CR/LF is rejected to block `$()` / backtick / newline command injection.
- **Symlink and privileged-run protection (CWE-59 / P2-2)**: Symlinked profiles and markers are rejected to prevent cross-path tampering; profile mutation is refused under privileged runs (administrator / root) to avoid writing to the wrong user's config.
- **Marker identity binding (F-4)**: The NodeCA marker is bound to the cert path and user identity; both write and read verify identity consistency, preventing cross-user marker misuse.
- **Fail-closed validation chain (F-1 / F-2)**: User-custom values in pwsh / POSIX profiles are checked before environment mutation — if present, `setx` / `launchctl` is skipped to avoid overwriting; profile parent-chain and filesystem-root validation failures, as well as profile read errors, fail closed; partial pwsh-profile failures return `ErrPartialSuccess` instead of being masked by any single success.

### Changed
- **Windows environment-change broadcast**: After `setx` writes, environment-variable changes are broadcast so running processes can observe the new `NODE_EXTRA_CA_CERTS`; the Windows client restart-sequence guidance is updated accordingly.
- **Legacy pwsh profile auto-migration**: On upgrade, legacy `$mccCa` residue lines are detected and cleaned, migrating the profile to the mcc marker block in one pass so the marker + `setx` persistence path resumes execution.

### Docs
- Node.js client CA-trust auto-setup feature spec and review notes (bilingual): `sdd-docs/features/2026-07-04-node-extra-ca-certs-auto-setup/`
- Design / plan / review documents for Node CA persistence correctness fixes, Windows environment refresh, and pwsh profile migration.
