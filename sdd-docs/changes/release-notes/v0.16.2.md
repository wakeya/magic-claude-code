## v0.16.2 Release Notes

### Fixed
- **Per-platform Release packages now bundle the full set of host setup scripts and docs**: Fixes the packaging step bundling only `setup-host.sh` / `setup-host.ps1` regardless of platform, which left Windows packages without `start-mcc.ps1`, `stop-mcc.ps1`, `register-mcc-task.ps1`, non-Windows packages without `docker-host-helper.sh`, and all packages without `README.en.md`, `SCRIPTS.md`, `SCRIPTS.en.md` (#12). Packages now include platform-matched scripts and bilingual docs so users can manually complete hosts + CA setup and container-host maintenance when bootstrap auto-configuration fails.
- **Eliminated an intermittent `SQLITE_BUSY` flaky in the providerquota scheduler test under `-race`**: The test DB previously ran in rollback-journal mode with no `busy_timeout`, so `scanAndQuery`'s synchronous `store.Get` raced an async `SaveUpsert` on the write lock and returned `SQLITE_BUSY` immediately, intermittently failing `TestSchedulerPeriodicScanNoJitter` in CI (run 29314320010). The test DSN now mirrors production (WAL + `synchronous(NORMAL)` + `busy_timeout(5000)`) with an added concurrency regression test. This is a test / CI quality fix with no runtime behavior change.
