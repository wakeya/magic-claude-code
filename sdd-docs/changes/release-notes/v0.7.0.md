## v0.7.0 更新内容

### Security
- **TLS 握手异步化与并发防护**：代理服务将 TLS 握手从同步模式改为异步并发模式，每个连接握手在独立 goroutine 中执行，设 10 秒超时和 100 并发上限，防止慢握手攻击导致服务饥饿
- **count_tokens 请求体大小限制**：通过 `io.LimitReader` 限制请求体大小，超过阈值返回 413，修复无限制读取导致 OOM 的 DoS 漏洞

### Added
- **Desktop 更新探测**：新增 `/api/desktop/**/update` 端点处理，HEAD 返回 200、GET 返回 currentRelease 版本号
- **事件日志 v2**：新增 `/api/event_logging/v2/batch` 端点
- **独立端点处理**：`policy_limits` 和 `remote_settings` 从空响应升级为结构化响应
- **bootstrap 字段补充**：响应增加 `cwk_cfg_key` 字段

### Changed
- **证书链完整性修复**：`SaveServerCert` 现在写入完整证书链（叶子证书 + CA 证书），修复客户端无法验证证书链的问题
- **品牌标识更新**：CA 名称从 "Claude Proxy" 改为 "MCC Proxy"，前端 logo 替换为品牌图片
- **请求日志增强**：日志增加 Host 字段，便于调试路由问题
- **静态资源白名单扩展**：admin 中间件新增 `.png`/`.ico`/`.svg` 免认证路径

### Fixed
- TLS 慢握手导致 listener starvation
- count_tokens 无限制请求体读取导致 OOM
- 证书链不完整导致部分客户端验证失败

## v0.7.0 Release Notes

### Security
- **Async, concurrency-guarded TLS handshake**: The proxy moved TLS handshakes from synchronous to async-concurrent mode; each connection's handshake runs in its own goroutine with a 10s timeout and a 100-connection cap, preventing slow-handshake attacks from starving the service
- **count_tokens request-body size limit**: `io.LimitReader` caps the request body and returns 413 when exceeded, fixing an OOM DoS from unbounded reads

### Added
- **Desktop update probe**: New `/api/desktop/**/update` endpoint handling — HEAD returns 200, GET returns the currentRelease version
- **Event logging v2**: New `/api/event_logging/v2/batch` endpoint
- **Standalone endpoint handling**: `policy_limits` and `remote_settings` upgraded from empty responses to structured responses
- **Bootstrap fields**: Added `cwk_cfg_key` to responses

### Changed
- **Certificate-chain completeness**: `SaveServerCert` now writes the full chain (leaf + CA), fixing clients that failed chain verification
- **Brand identity**: CA name changed from "Claude Proxy" to "MCC Proxy"; frontend logo replaced with the brand image
- **Request logs**: Added Host field for routing debugging
- **Static-asset allowlist**: admin middleware adds `.png`/`.ico`/`.svg` unauthenticated paths

### Fixed
- Slow TLS handshakes caused listener starvation
- Unbounded count_tokens request reads caused OOM
- Incomplete certificate chain failed verification on some clients
