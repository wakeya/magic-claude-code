## v0.8.0 更新内容

### Security
- **URL 凭证脱敏**：`RedactURL` 现在剥离 `https://user:pass@host` 形式的 userinfo；代理入口/出口日志、usage 读取路径统一走脱敏，防止 provider URL 中的凭证或签名泄露到日志和管理 API
- **usage 读取二次脱敏**：防御历史脏数据，即使旧记录存了未脱敏 URL，读取时也统一 redact，Coverage/Requests 两条输出路径均不泄露

### Added
- **透明模式自动引导**：启动时自动尝试 hosts 修改、CA 信任安装、MCC_ROOT 环境持久化；引导失败不阻塞代理启动，按优先级降级
- **三连接模式与自动降级**：透明模式 > 隧道模式 > 网关模式，按优先级自动选择；前端 header 模式按钮可切换并持久化到后端，`/api/config` 和 `/api/status` 暴露首选/实际模式
- **i18n 系统语言检测**：`zh*` locale 默认中文，其他默认英文；支持 `MCC_LANG` 环境变量手动覆盖
- **首运行 MCC_ROOT 持久化**：首次运行将可执行文件目录持久化到 shell profile 或 Windows 环境变量，后续从任意工作目录启动均可自动定位证书
- **fish shell profile 去重增强**：fish 导出行匹配更语义化（导出标志识别、注释剥离、引号/转义处理、scalar vs list 区分），避免重复追加
- **Docker 宿主机 helper 机制**：`MCC_HOST_HELPER` 环境变量支持挂载宿主机 helper 脚本，容器通过 helper 检测/修改宿主机 hosts 与 CA 信任
- **宿主机一键配置脚本**：新增 `scripts/setup-host.sh`（Linux/macOS）、`scripts/setup-host.ps1`（Windows），支持 `hosts`/`trust` 子命令与 helper 接口；`scripts/docker-host-helper.sh` 作为容器内默认状态检测器
- **docker-compose 部署**：新增 `docker-compose.yml`，集成端口映射、数据卷、usage 同步目录与 NET_BIND_SERVICE 能力
- **前端模式入口与弹窗**：header 新增连接模式入口区域和三模式说明弹窗，含 i18n 文案
- **CI 测试工作流**：新增 `.github/workflows/test.yml`，push/PR 时运行 `make test`（含 race detector）
- **发布资产附带宿主机脚本**：Release archive 附带 `setup-host.sh`/`setup-host.ps1`，bootstrap 失败时用户可手动完成 hosts + CA 配置

### Changed
- **请求日志增强**：入口日志延后到 backendURL 确定后打印，附带 `provider_name` 与脱敏后的 `upstream_url`；出口日志同步增加 provider/upstream 字段；rate-limit 日志改用 `provider_name`（名称）替代 `provider`（ID）
- **bootstrap 结果模型**：hosts/CA/环境持久化三项独立记录，不再合并为单一错误字符串；bootstrap 状态持久化到 data 目录，抑制重复失败日志
- **`AGENT.md` 重命名为 `AGENTS.md`**：对齐主流 agent 文件约定
