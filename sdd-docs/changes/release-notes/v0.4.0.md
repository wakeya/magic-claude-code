## v0.4.0 更新内容

### Added
- **供应商级主动内容块清理**：新增 `strip_unknown_content_blocks` 开关（默认关闭），在转发请求前主动移除非标准内容块（如 `tool_reference`），解决 Kimi 等严格校验端点返回 400 错误的问题
- 兼容性错误识别增强：rectifier 新增匹配 `unsupported/unknown content type` 错误模式，扩大被动修复覆盖范围
- 前端供应商弹窗新增内容块清理开关 UI，支持中英文

### Changed
- Gitee 发布同步默认关闭，需设置 `ENABLE_GITEE_SYNC=true` 变量启用
- Gitee 发布流水线强制使用 bundled Go toolchain，修复构建环境差异

### Docs
- 新增提交与发布工作流指南（AGENT.md、CLAUDE.md、README.md）

## v0.4.0 Release Notes

### Added
- **Provider-level proactive content-block cleanup**: New `strip_unknown_content_blocks` toggle (default off) proactively removes non-standard content blocks (e.g. `tool_reference`) before forwarding requests, resolving 400 errors from strict validators like Kimi
- Compatibility-error detection enhanced: the rectifier now matches the `unsupported/unknown content type` error pattern, broadening reactive-recovery coverage
- Frontend provider modal adds a content-block cleanup toggle (i18n)

### Changed
- Gitee release sync is now off by default; set `ENABLE_GITEE_SYNC=true` to enable
- The Gitee release pipeline forces the bundled Go toolchain, fixing build-environment drift

### Docs
- Added commit-and-release workflow guide (AGENT.md, CLAUDE.md, README.md)
