## v0.9.2 更新内容

### Fixed
- 管理面板标签页切换布局抖动：
  - **滚动条 reflow 修复**：`html { scrollbar-gutter: stable; overflow-y: auto }` 为垂直滚动条预留稳定 gutter，切换含长内容的 tab（如 会话记录）时顶部状态栏和标签块不再横向抖动
  - **会话记录激活重绘修复**：会话列表数据（`getSessionProjects` + `getSessionList`）从 `SessionBrowser.onMounted` 上提到 `DashboardView.onMounted` 预加载，与其他五个 tab 对齐；激活会话记录时列表即时渲染，消除异步空→满二次布局
  - **骨架屏**：sessions 列表区新增 `animate-pulse` 骨架占位（`min-h-[28rem]`），替代原纯文字 loading，兜底首次加载和刷新
  - **加载失败错误态**：sessions 列表加载失败时优先显示错误信息，不再误显示"无会话"空态

## v0.9.2 Release Notes

### Fixed
- Admin panel tab-switch layout jitter:
  - **Scrollbar reflow fix**: `html { scrollbar-gutter: stable; overflow-y: auto }` reserves a stable gutter for the vertical scrollbar, so the top status bar and tab blocks no longer shift horizontally when switching to long-content tabs (e.g. session records)
  - **Session-records active-reflow fix**: Session-list data (`getSessionProjects` + `getSessionList`) is hoisted from `SessionBrowser.onMounted` to `DashboardView.onMounted` for preloading, aligning with the other five tabs; activating session records renders the list immediately, eliminating the async empty→full second layout
  - **Skeleton screen**: The sessions list area adds an `animate-pulse` skeleton placeholder (`min-h-[28rem]`), replacing the plain-text loading, covering first load and refresh
  - **Load-failure error state**: When the sessions list fails to load, the error message is shown first, instead of misleadingly showing an empty "no sessions" state

- Admin-panel tab-switch layout-jitter fix spec (bilingual, 4/4 tasks done and verified): `sdd-docs/features/2026-06-24-dashboard-tab-layout-shift/`
