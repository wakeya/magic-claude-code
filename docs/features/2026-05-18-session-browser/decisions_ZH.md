# Claude Code 会话记录浏览器设计决策

## D-001：读取本地文件取代代理拦截

**日期：** 2026-05-20
**状态：** 已采纳

**背景：** 原始 v0.2 方案通过拦截代理流量捕获对话，重建 SSE 流，用 fingerprint 分组会话。需要修改代理处理器、构建 SSE 观察器、猜测会话边界。

Claude Code 已将完整会话记录以 JSONL 文件形式存储在 `~/.claude/projects/` 下，包含原生 sessionId、项目路径和自定义标题。

**决策：** 直接从本地文件系统读取 JSONL 文件。无需修改代理。

**影响：** 消除了 SSE 观察器、normalizer、fingerprint 和隐私 opt-in 的复杂性。会话分组准确。要求 Claude projects 目录可访问（本地默认 `~/.claude/projects/`，Docker 通过 `CLAUDE_PROJECTS_DIR` 挂载或覆盖）。

**重新审视时机：** 多实例部署需要聚合不同机器的会话时。

## D-002：项目目录作为会话筛选维度

**日期：** 2026-05-20
**状态：** 已采纳

**背景：** cc-switch 以扁平列表加 provider 过滤器展示会话。项目目录仅作为每张会话卡片上的元数据展示，不参与定位会话。

Claude Code 在文件系统布局中按项目目录组织会话（`~/.claude/projects/<编码路径>/`）。用户自然地在项目语境下思考会话。

**决策：** 项目是会话列表的一级筛选维度。选择项目后展示该项目会话；`"全部项目"` 选项展示跨项目的所有会话。具体布局由 D-006 定义为左侧项目下拉选择器加会话列表。

**影响：** UI 保持清晰的两级关系（项目筛选 → 会话列表）。用户可快速定位特定项目的会话，无需在无关会话中滚动。

**重新审视时机：** 用户项目过多，需要项目分组（如按父目录或工作空间）时。

## D-003：直接文件读取配合内存缓存

**日期：** 2026-05-20
**状态：** 已采纳

**背景：** 会话数据存在于不受我们控制的 JSONL 文件中。性能方案选项：(a) 每次 API 调用都读文件，(b) SQLite 索引，(c) 带 TTL 的内存缓存。

**决策：** 按需读取文件，配合内存缓存（TTL 约 30 秒）。列表用途仅读取每个 JSONL 文件的头部/尾部行。详情用途解析完整文件。

**影响：** 比维护 SQLite 索引简单。无同步问题。缓存 TTL 意味着列表可能略有延迟，但手动刷新始终可用。对单用户管理面板足够。

**重新审视时机：** 用户反馈 1000+ 会话下列表缓慢，或需要全文搜索时。

## D-004：服务端 HTML 渲染用于导出

**日期：** 2026-05-20
**状态：** 已采纳

**背景：** 导出的 HTML 需要离线工作，无外部依赖。方案选项：(a) Go 模板服务端渲染，(b) 客户端 JS 生成。

**决策：** 使用 `html/template` 在服务端渲染 HTML。导出文件包含内联 CSS、服务端渲染的消息和极简 JS 用于折叠切换。

**影响：** 导出在禁用 JavaScript 时也能工作。Go 标准库即可实现，简单直接。v1 无需 Markdown 库——纯文本加基本格式足够。

**重新审视时机：** 用户需要在导出 HTML 中支持富 Markdown 渲染（代码高亮、表格、链接）时。

## D-005：管理面板不删除 Claude Code 会话文件

**日期：** 2026-05-20
**状态：** 已采纳

**背景：** Claude Code 会话 JSONL 和附属目录属于 Claude Code 自身状态。管理面板直接删除这些文件会带来误删、路径逃逸、符号链接和并发写入风险，也可能破坏 Claude Code 对项目状态的管理假设。

**决策：** 会话浏览器不提供删除 JSONL 文件或附属目录的 API，也不在前端执行删除操作。需要清理项目或会话状态时，界面仅展示 Claude Code CLI 命令提示。项目级提示优先使用 `claude project purge --dry-run <project-path>` 预览，再提示用户在终端自行运行 `claude project purge -i <project-path>` 进行交互确认。单会话清理以当前 Claude Code 版本的 CLI 帮助为准。

**影响：** 功能保持只读，降低破坏性操作风险。用户仍可通过 Claude Code 官方命令行完成清理，但管理面板不承担文件删除职责。

**重新审视时机：** Claude Code 提供稳定、受支持的单会话删除 CLI/API，并明确其文件和索引一致性语义时。

## D-006：左侧项目下拉与会话列表，右侧展示会话详情

**日期：** 2026-05-21
**状态：** 已采纳

**背景：** 会话记录页面存在两种可行结构：

1. 左侧只显示项目，右侧在“会话列表页”和“会话详情页”两层之间切换。
2. 左侧顶部使用项目下拉选择器，左侧主体展示当前项目会话列表，右侧始终展示会话详情。

第一种结构让项目导航更纯粹，但用户进入详情后会话列表消失，切换同一项目下的其他会话需要先返回列表。会话浏览的高频操作是反复查看和切换会话，因此保持列表可见更重要。

**决策：** 采用第二种结构。左侧顶部提供项目下拉选择器，支持 `"全部项目"` 和具体项目；左侧主体展示当前项目的会话列表并独立滚动；右侧展示当前选中会话的详情、导出、清理提示和用户消息大纲。

**影响：** 用户可在同一项目内快速切换会话，不需要返回按钮。左侧面板职责从“项目树”变为“项目筛选 + 会话导航”，右侧面板职责保持为详情阅读和详情操作。

**重新审视时机：** 项目数量或会话数量过大，单一下拉选择和左侧滚动列表无法高效定位时，可增加项目搜索、会话搜索或列表虚拟滚动。
